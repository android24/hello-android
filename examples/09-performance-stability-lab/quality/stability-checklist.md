# 稳定性体检清单

## 崩溃

- [ ] 可预期错误使用 Result、sealed class 或明确错误状态建模。
- [ ] 崩溃报告包含堆栈、版本、设备、系统版本和用户路径。
- [ ] 关键业务路径有错误兜底。
- [ ] release 包可以还原混淆后的堆栈。

## ANR

- [ ] 主线程没有等待锁。
- [ ] 主线程没有同步等待后台结果。
- [ ] 广播、Service 和启动链路没有长时间阻塞。
- [ ] 可以区分 CPU 忙、锁等待、IO 等不同 ANR 原因。

## 日志

- [ ] debug 包可以输出详细诊断日志。
- [ ] release 包关闭 verbose 日志。
- [ ] 日志不包含 token、手机号、邮箱等敏感信息。
- [ ] 关键失败场景有可定位的错误码或上下文。

## 发布观察

- [ ] 发布后观察崩溃率。
- [ ] 发布后观察启动失败和接口错误。
- [ ] 关键路径异常有告警。
- [ ] 严重问题有回滚或降级方案。
